# 국민안전24 연동 — 내가 할 일 체크리스트

`daily-fire-dashboard-briefing` 스케줄은 서지나님 개인 Claude 계정에 있어서, 이 세션에서 제가 대신 수정할 수 없다.
아래는 개발자에게 개발가이드를 넘기는 것과 별개로, **서지나님이 직접 처리해야 하는 부분**만 모은 것이다.

## 체크리스트

- [ ] **1. 개인 계정에서 스케줄 프롬프트 수정**
  Settings(또는 스케줄 관리 화면)에서 `daily-fire-dashboard-briefing` 스케줄을 열고, 아래 "복붙용 추가 문구"를 프롬프트에 넣는다. (한 세션 열어서 저한테 "이거 추가해줘"라고만 해도 됨 — 그 계정 세션에서 `update_scheduled_task`로 바로 반영 가능)

- [ ] **2. Claude in Chrome 확장 연결 유지**
  스케줄이 국민안전24/소방청 페이지를 수집하려면 매일 실행 시점에 Chrome 확장이 연결돼 있어야 한다. 연결이 끊기면 그날은 "수집 실패"로 보고되고 블루스캔 앱은 자동으로 목업 데이터로 대체 표시된다(에러 없이 조용히 대체됨 — 그래서 며칠에 한 번은 실제로 갱신되고 있는지 확인 권장).

- [ ] **3. `fireDispatch.json` 공개 여부 승인**
  기존 `midoseo/fire_dashboard` 저장소(→ Netlify 공개 배포)에 시도별 통계 JSON을 함께 올리는 구조다. 개인정보·민감정보는 없는 집계 수치이므로 공개해도 되는지만 한 번 확인.

- [ ] **4. 개발자에게 전달**
  - `화재_국민안전24연동_개발가이드.md` (개발 파일 변경 내역)
  - `app/src/fireDispatch.demo.js` (기대하는 데이터 구조 샘플)
  - 수정된 `SKILL.md` (1번 항목 반영 후 버전)
  - Netlify 배포 URL: `https://loquacious-semifreddo-e21627.netlify.app/`
  - `midoseo/fire_dashboard` 저장소 접근 권한(또는 최소 열람 권한)

- [ ] **5. 개발 완료 후 확인**
  - 다음날 스케줄 실행 후 `fireDispatch.json`이 실제로 갱신됐는지 (Netlify URL로 직접 열어서 오늘 날짜인지 확인)
  - 블루스캔 앱을 실행해 대시보드의 "오늘의 화재 출동 현황" 숫자가 목업이 아닌 실제 값으로 바뀌었는지 확인

## 복붙용 추가 문구 (스케줄 프롬프트에 그대로 삽입)

기존 SKILL.md의 "4. 대시보드 HTML 갱신"과 "5. GitHub 배포" 사이에 아래 단계를 추가한다.

```
## 4-1. 화재 출동 통계 JSON 저장 (블루스캔 앱 연동용)
- 1단계에서 이미 수집한 시도별 접수(FIRE_RCPT_MNB)/진행(FIRE_PROG_MNB)/인명피해(LIFE_DMG_PERCNT)/재산피해(PRPT_DMG_SBTT_AMT)
  데이터를 재사용해, outputs에 fireDispatch.json을 아래 구조로 저장한다. 새로 수집할 데이터는 없음 — 포맷만 바꾸는 단계.
  {
    "isMock": false,
    "source": "소방청 화재정보 공개API(국민안전24)",
    "baseDate": "{어제 YYYY-MM-DD}",
    "national": { "rcpt": 전국합계접수, "prog": 전국합계진행, "life": 전국합계인명피해, "prop": 전국합계재산피해 },
    "bySido": [ { "sido": "경기", "rcpt": 0, "prog": 0, "life": 0, "prop": 0 }, ... ]   // 접수 건수 내림차순 정렬
  }
- 필드명(isMock/source/baseDate/national/bySido, 그리고 national·bySido 하위의 rcpt/prog/life/prop)은 블루스캔 앱이
  그대로 읽어가므로 절대 바꾸지 않는다.
```

기존 "5. GitHub 배포" 단계 문구 중 업로드 부분을 아래처럼 바꾼다.

```
- Claude in Chrome으로 https://github.com/midoseo/fire_dashboard/upload/main 이동 → 파일 입력(ref)에
  outputs의 index.html과 fireDispatch.json 둘 다 file_upload → 커밋 메시지 "Update dashboard {오늘YYYY-MM-DD}" →
  "Commit changes" 클릭(기존 덮어쓰기). index.html·fireDispatch.json만 올리고 키/지침은 올리지 않는다.
```
