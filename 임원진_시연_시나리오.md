# 임원진 시연 시나리오 (5분 버전)

**컨셉**: "컨설턴트 윤연연 님의 월요일 아침"을 5분 안에 압축 — 신규 발굴 한 건, 유지고객 대응 한 건만 처리하면서 핵심 차별점(데이터 기반 우선순위 → AI 분석 → 재무 기반 리텐션 판단 → 게이미피케이션)만 짚는다. 업셀링 리스트·월간 리포트 PDF·시즌 토글 데모는 시간상 뺐고, 질문 나오면 구두로만 언급한다(부록 참고). **12~15분 분량의 심화판은 이 문서 아래쪽에 그대로 남겨뒀다.**

## 시연 전 준비
- 인터넷 연결 필수(지도 타일·아이콘 폰트·화재 데이터가 외부에서 로드됨)
- 브라우저 새로고침 한 번 해서 최신 배포본인지 확인
- 로그인 화면에서 아무 것도 입력하지 말고 **"발표·시연용 추천 계정" 카드의 "이 계정으로 바로 로그인"**만 누르면 됨(사번 0232931336 자동 입력됨)

## 시간 남으면 하나만 추가
- 지도에서 "화재 오버레이"·"침수 오버레이" 칩 켜서 겹쳐 보여주기(15초)
- "대성보건소" → "월간 리포트 보기" → "PDF로 다운로드" (25초, 실제 파일 다운로드까지 보여주고 싶을 때)

## 부록 — 질문 대비 (구두로만 언급)
- 업셀링 리스트(B-1/B-2), 시즌 자동 전환(계절 토글)은 시간상 생략 — 물어보면 "같은 원리로 기존 고객 리스트에도 놓친 신호를 띄워준다"고 답변
- **리더보드 동료 점수는 예시 데이터**(본인 점수만 실제 활동 기반) — 실전환 시 동료도 실데이터로 교체 예정
- 화재·침수·신호 이력은 **비식별 예시 데이터**
- 이탈 스코어·등급은 **의도적으로 미구현** — "예측 모델 기준은 사람이 설계해야 하는 영역이라 신호 표시 단계까지만 구현했다"고 답변(과장하지 않았다는 인상)
- 막히면 새로고침 후 데모 계정 재로그인(포인트·방문기록만 초기화, 나머지 데이터는 그대로)

# 심화판 (12~15분 — 질의응답·후속 미팅용)

## 1. 로그인 → 컨설턴트 대시보드 훑기 (2분)

| 조작(입력) | 확인할 출력 | 멘트 포인트 |
| --- | --- | --- |

| 화면 위쪽 "영업 알림" 배너 훑기 | "[혹서기·풍수해기] 침수 위험구역 18곳 우선영업", "경비원 계약 만료 임박 13곳" | "타이밍 신호가 알아서 알림으로 뜹니다" |
| 상단바 우측 "50P" 포인트 배지 짚기 | 숫자만 확인 | "이후 활동할 때마다 이 숫자가 실시간으로 오릅니다 — 잠시 후 보여드릴게요" |
| "블루스캔 관련 뉴스" 카드 확인 | 오늘의 화재 출동 현황(접수·진행·인명·재산피해) + 실시간 스크롤 리스트(마포구 합정동·망원동·공덕동 화재) | "소방청·국민안전24 공개데이터를 그대로 붙였습니다" |

## 부록 — 시연 전 꼭 확인할 것 / 질문 대비

- **리더보드의 동료 점수는 예시 데이터**입니다(본인 점수만 실제 활동 기반). 질문 나오면 "실전환 시 동료도 실데이터로 교체 예정"이라고 답변.
- 화재·침수 데이터, 신호 이력 등은 **비식별 예시 데이터**입니다(발표 중 "실제 고객 데이터냐"는 질문 대비).
- 이탈 스코어·등급(예: "이탈 위험도 80점")은 **의도적으로 넣지 않았습니다** — "예측 모델 기준은 사람이 설계해야 하는 영역이라, 신호를 그대로 보여주는 단계까지만 구현했다"고 설명하면 오히려 신뢰도가 올라감(과장하지 않았다는 인상).
- PDF 다운로드는 이미지 기반이라 텍스트 선택이 안 됨 — 굳이 언급할 필요는 없지만 파일을 열어서 "고객사에 바로 보낼 수 있는 완성본"이라는 점만 보여주면 충분.
- 시연 중 막히면 즉시 새로고침 후 데모 계정으로 재로그인하면 방문 결과·포인트는 초기화되지만(세션 기준) 나머지 데이터는 그대로.
